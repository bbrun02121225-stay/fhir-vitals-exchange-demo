# 會議記錄（Bullet Notes）

## 會議一  
**日期：2024/12/17**  
**主題：FHIR 實作方向與程式呈現方式討論**

- 目前進度  
  - 已使用 Python 撰寫一支簡單程式  
  - 功能包含：
    - POST Patient（建立病人）
    - POST Observation（上傳生命徵象）
    - GET Patient / Observation（讀取資料）
  - 主要用於驗證 FHIR Server 可正常寫入與讀取資料  

- 會議說明內容  
  - 說明 Python 程式設計理念  
  - 強調以 FHIR API 為核心進行資料交換  
  - 示範基本資料上傳與查詢流程  

- 教授回饋  
  - 技術實作方向正確  
  - 目前成果偏向工程測試（PoC）  
  - 不符合「使用者可實際操作」的實作導向期待  
  - 建議：
    - 增加前端操作介面  
    - 將 Python 程式邏輯轉為 HTML + JavaScript  

- 會議結論  
  - Python 程式作為概念驗證保留  
  - 正式競賽成果需改為前端系統呈現
